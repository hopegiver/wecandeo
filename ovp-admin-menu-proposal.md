## 📌 핵심 워크플로우

```
1. 업로드
   ↓ (자동 기본 인코딩 시작)

2. 미디어 보관함 (Media Library)
   ↓ (원본 + 인코딩 버전 보관)

3. 배포 패키지 (Distribution Package)
   ↓ (필요시 추가 인코딩: DRM, 워터마크 등)

4. 배포 룰 설정 (Delivery Rules)
   - 접근 제어 (도메인, IP, 토큰 인증)
   - 플레이어 스킨
   - Geo-blocking
   ↓

5. 배포 (Publish)
```

### 2. 📹 미디어 (Media)

#### 2.1 미디어 보관함 (Media Library)
**모든 동영상 원본 및 인코딩 버전 관리**

핵심 기능:
- 그리드/리스트 뷰 전환
- 고급 필터 (상태, 날짜, 태그, 카테고리)
- 일괄 작업 (태그 추가, 이동, 삭제)
- 드래그 앤 드롭 업로드
- **인코딩 상태 표시** (대기중/진행중/완료/실패)
- 미리보기 플레이어

```
┌─────────────────────────────────────────┐
│ video.mp4          [배포 패키지에 추가]   │
├─────────────────────────────────────────┤
│ 상태: ✅ 인코딩 완료                      │
│ 원본: 1920x1080 (500MB)                 │
│ 버전: 1080p, 720p, 480p                 │
│ 태그: #마케팅 #제품소개                  │
└─────────────────────────────────────────┘
```

#### 2.2 업로드
**Brightcove 스타일의 스마트 업로드**

- 단일/다중/폴더 업로드
- 대용량 resumable upload (청크 업로드)
- **CSV/XML 일괄 업로드** (Kaltura 방식)
  - 메타데이터 포함 대량 등록
- 업로드 시 인코딩 옵션 선택
  - ✅ 자동 인코딩 (권장): 1080p, 720p, 480p
  - ⭐ 선택 인코딩: 4K, 360p 추가 가능
  - ⏸️ 나중에 인코딩
- 진행 상태 모니터링 (큐 관리)

#### 2.3 폴더 & 카테고리
**계층적 콘텐츠 구조**

- 폴더 트리 구조
- 카테고리 태깅
- 자동 분류 규칙 (AI 태깅)

### 3. 📦 배포 (Distribution)

#### 3.1 배포 패키지 (Packages)
**Brightcove Social Syndication + Kaltura Distribution 개념 통합**

핵심 개념:
- 미디어 보관함의 동영상을 배포 패키지에 추가
- 배포 패키지는 배포 룰을 가짐
- 하나의 동영상이 여러 배포 패키지에 포함 가능

주요 기능:
- 패키지 생성/편집/삭제
- 동영상 추가/제거
- 패키지별 배포 룰 연결
- 배포 상태 관리 (활성/비활성/예약)

```
┌─────────────────────────────────────┐
│ 패키지: "고객용 제품 소개"            │
├─────────────────────────────────────┤
│ 포함 동영상: 15개                    │
│ 배포 룰: 기본 공개 룰                │
│ 상태: ✅ 활성                        │
│ [동영상 추가] [룰 변경] [비활성화]    │
└─────────────────────────────────────┘
```

#### 3.2 배포 룰 (Delivery Rules)
**접근 제어 + 플레이어 설정의 조합**

하나의 배포 룰 = 보안 정책 + 플레이어 스타일

**보안 설정:**
- 도메인 제한 (화이트리스트/블랙리스트)
- IP 주소 제한
- 지역 차단 (Geo-blocking)
- 토큰 인증 (Signed URL)
- DRM (Widevine, FairPlay, PlayReady)
- 비밀번호 보호
- 시간 제한 링크

**플레이어 설정:**
- 플레이어 스킨 선택
- 로고/워터마크
- 컨트롤 UI 커스터마이징
- 자동재생, 반복재생 옵션
- 공유 버튼 표시 여부

**외부 배포:**
- YouTube 자동 동기화
- 소셜 미디어 연동
- RSS 피드 생성

#### 3.3 임베드 코드
**간편한 코드 생성기**

- iframe 코드 자동 생성
- 반응형/고정 크기 선택
- 옵션 프리뷰

## 💡 우리의 차별화 포인트

### 1. 더 쉬운 배포 개념
**기존**: "어디에 배포? 어떻게 배포?"
**개선**: "배포 패키지 만들고 → 룰 선택 → 끝"

### 2. 스마트 인코딩
- 업로드 시 자동 기본 인코딩 (빠른 배포)
- 배포 룰에 따른 추가 인코딩 (DRM, 워터마크)
- AI 기반 최적 프로필 추천

### 3. 통합 배포 룰
**기존**: 보안 설정, 플레이어 설정이 분리
**개선**: 하나의 "배포 룰"에 모두 통합
- "공개용", "내부용", "프리미엄" 등 프리셋
- 재사용 가능, 쉬운 적용

### 4. 시각적 워크플로우 빌더
Kaltura의 자동화를 더 쉽게
- 드래그 앤 드롭으로 규칙 생성
- IF-THEN 조건 시각화

## 📊 메뉴 구조 비교: VideoPack vs Brightcove vs Kaltura

### 전체 메뉴 비교표

| 기능 영역 | VideoPack | Brightcove Video Cloud | Kaltura KMC | 특징 |
|---------|-----------|----------------------|-------------|------|
| **대시보드** | ✅ 홈 (통합 대시보드) | ✅ Dashboard | ✅ Dashboard | VideoPack: 인코딩 현황 실시간 표시 |
| **콘텐츠 관리** | ✅ 미디어 (3개 하위메뉴) | ✅ Media | ✅ Content | VideoPack: 그리드/리스트 뷰 통합 |
| **업로드** | ✅ 미디어 > 업로드 | ✅ Upload | ✅ Upload | VideoPack: 인코딩 옵션 선택 UI |
| **배포/게시** | ✅ 배포 (패키지+룰) | ✅ Social/Destinations | ✅ Distribution | VideoPack: **통합 배포 룰 개념** ⭐ |
| **인코딩** | ✅ 제작 > 트랜스코딩 | ✅ Admin > Ingest Profiles | ✅ Settings > Transcoding | 모두 유사 |
| **자막** | ✅ 제작 > 자막 & 캡션 | ✅ Media > Captions | ✅ Entry > Captions | 모두 유사 |
| **플레이어** | ✅ 설정 > 플레이어 스튜디오 | ✅ Players | ✅ Studio | Kaltura 방식 채택 |
| **분석** | ✅ 분석 (5개 하위메뉴) | ✅ Analytics | ✅ Analytics | Brightcove 구조 + Kaltura 상세도 |
| **수익화** | ✅ 수익화 (광고/구독) | ✅ Advertising | ✅ Monetization | Brightcove SSAI + Kaltura 모델 |
| **권한 관리** | ✅ 사용자 > 팀 관리 | ✅ Admin > Users | ✅ Administration > Users | Kaltura RBAC 방식 |
| **자동화** | ✅ 시스템 > 자동화 | ❌ 없음 | ✅ Settings > Rules | **Kaltura 독점 기능 채택** ⭐ |
| **라이브** | ✅ 시스템 > 라이브 | ✅ Live | ✅ Live Stream | 모두 지원 |
| **API** | ✅ 설정 > API & Webhooks | ✅ Admin > API Keys | ✅ Settings > API | 모두 유사 |

### 주요 차이점 분석

#### 1. 메뉴 구조 철학

**Brightcove Video Cloud**
```
Media (콘텐츠 중심)
├── Videos
├── Playlists
├── Live
└── Social

Players (플레이어 관리)
Analytics (분석)
Admin (설정)
```
- **장점**: 단순하고 직관적, 콘텐츠 중심
- **단점**: 고급 기능 접근이 어려움, 배포 개념 분산

**Kaltura KMC**
```
Content (콘텐츠)
├── Entries
├── Categories
├── Playlists
└── Moderation

Studio (플레이어)
Settings (설정)
├── Transcoding
├── Distribution
├── Access Control
└── Metadata

Analytics (분석)
Administration (관리)
```
- **장점**: 엔터프라이즈급 세밀한 제어, 강력한 자동화
- **단점**: 복잡한 구조, 학습 곡선 높음

**VideoPack OVP (우리)**
```
홈 (대시보드)

미디어 (콘텐츠 라이프사이클)
├── 미디어 보관함
├── 업로드
└── 폴더 & 카테고리

배포 (배포 중심) ⭐ 차별화
├── 배포 패키지
├── 배포 룰
└── 임베드 코드

제작 (후처리)
├── 트랜스코딩
├── 자막 & 캡션
└── 편집 도구

분석 (데이터 중심)
수익화 (비즈니스)
설정 (기술 설정)
사용자 (권한)
시스템 (자동화 & 라이브)
```
- **장점**:
  - 워크플로우 기반 구조 (업로드 → 제작 → 배포 → 분석)
  - 배포 개념 단순화 (패키지 + 룰)
  - Brightcove 직관성 + Kaltura 강력함
- **차별점**:
  - **통합 배포 룰**: 보안 + 플레이어를 하나로
  - **워크플로우 중심**: 실제 작업 순서대로 메뉴 배치

#### 2. 구체적 기능 비교

##### 배포 기능 (가장 큰 차별점)

| 기능 | Brightcove | Kaltura | VideoPack |
|-----|-----------|---------|-----------|
| 배포 개념 | Social Destinations (분산) | Distribution Profiles | **배포 패키지 + 배포 룰** ⭐ |
| YouTube 연동 | ✅ Social | ✅ Distribution | ✅ 배포 > 외부 배포 |
| 접근 제어 | Admin > Settings | Settings > Access Control | **배포 룰에 통합** ⭐ |
| 플레이어 설정 | Players (별도) | Studio (별도) | **배포 룰에 통합** ⭐ |
| DRM | Admin > DRM | Settings > DRM | **배포 룰에 통합** ⭐ |

**VideoPack의 혁신**:
- 기존 플랫폼: 보안(Access Control), 플레이어(Player), 배포(Distribution)가 모두 **분리**
- VideoPack: 하나의 "배포 룰"에 **모두 통합**
  ```
  배포 룰: "공개용"
  ├── 보안: 도메인 제한 없음
  ├── 플레이어: 공개 플레이어 스킨
  └── 외부: YouTube 자동 동기화

  배포 룰: "내부용"
  ├── 보안: 회사 IP만 허용
  ├── 플레이어: 로고 워터마크
  └── 외부: 배포 안 함
  ```

##### 자동화 기능

| 항목 | Brightcove | Kaltura | VideoPack |
|-----|-----------|---------|-----------|
| 자동화 규칙 | ❌ 없음 | ✅ Rules Engine | ✅ 시스템 > 자동화 |
| 워크플로우 | ❌ 없음 | ✅ Workflow Builder | ✅ 워크플로우 빌더 |
| 조건부 액션 | ❌ API만 | ✅ UI에서 설정 | ✅ UI + 시각화 |
| 일괄 처리 | 제한적 | ✅ 500개 배치 | ✅ CSV/XML 일괄 |

**Brightcove의 약점**: 자동화 기능 부재 → API로만 해결
**Kaltura 장점 채택**: Rules Engine을 더 쉬운 UI로

##### 업로드 & 인코딩

| 기능 | Brightcove | Kaltura | VideoPack |
|-----|-----------|---------|-----------|
| 업로드 방식 | 단일/API | 단일/CSV/XML/API | **단일/다중/CSV/XML** ⭐ |
| 인코딩 타이밍 | 업로드 시 자동 | 선택 가능 | **스마트 인코딩** ⭐ |
| 프로필 선택 | 업로드 전 선택 | 업로드 후 설정 | **업로드 시 선택 가능** |
| 진행 상태 | 리스트에서 확인 | 별도 페이지 | **대시보드 통합** ⭐ |

**VideoPack 혁신**:
- 업로드 시 인코딩 옵션 선택 UI (Brightcove에는 없음)
- 홈 대시보드에 인코딩 현황 실시간 표시
- "자동 인코딩 (권장)" vs "나중에 인코딩" 명확한 선택지

#### 3. UX/UI 개선 사항

##### 정보 계층 구조

**Brightcove**: 너무 단순 (2단계)
```
Media > Videos (끝)
```

**Kaltura**: 너무 복잡 (4단계)
```
Settings > Transcoding > Flavor Params > Edit
```

**VideoPack**: 적절한 균형 (3단계)
```
제작 > 트랜스코딩 > [프로필 상세]
```

##### 용어 개선

| 기존 용어 | VideoPack | 이유 |
|---------|-----------|------|
| Ingest Profiles (Brightcove) | 인코딩 프로필 | 더 직관적 |
| Flavor Params (Kaltura) | 인코딩 프로필 | 전문 용어 제거 |
| Access Control (Kaltura) | 배포 룰 | 목적 중심 |
| Social Destinations (Brightcove) | 외부 배포 | 명확한 의미 |
